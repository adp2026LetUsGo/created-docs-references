## SECTION 1 — 7-CELL ARCHITECTURE

```
AHS.Cell.Xinfer is decomposed into 7 specialized Cells:

CELL-1: AHS.Cell.Xinfer.ShipmentInput
  Purpose: Capture intrinsic + environmental + operational shipment data
  Port: IShipmentInputPort
  Adapters: UIInputAdapter, APIInputAdapter, LocalInputAdapter (CSV)
  Output: ShipmentDTO

CELL-2: AHS.Cell.Xinfer.DataReadiness
  Purpose: Evaluate quality and predictability of new shipment data
  Validations (9):
    - Historically problematic carrier
    - Incompatible season
    - Route divergence
    - Dataset too small or too large
    - Thermal or temporal outliers
    - Incompatible packaging
    - New carrier/route never seen before
    - Abrupt climate changes
    - Missing values
  Output: ReadinessResult { Status, Errors[], Warnings[] }
          Status: Acceptable | NotAcceptable | Risky

CELL-3: AHS.Cell.Xinfer.HistoricalSelector
  Purpose: Auto-select correct historical data for model training
  Logic:
    - Same route
    - Same packaging
    - Similar season (±45 days)
    - Carrier: optional (user decision)
    - Minimum N shipments (configurable, default 10)
    - Maximum M shipments (configurable, default 200)
    - Eliminate outliers (IQR method, AOT-safe)
  Output: HistoricalDataset { Records[], Metadata }

CELL-4: AHS.Cell.Xinfer.RetrainDecider
  Purpose: Decide IF model should retrain (does NOT train)
  Criteria:
    - +N shipments since last retrain (configurable threshold)
    - Seasonal changes detected
    - New carrier or route added
    - Model degradation (accuracy below threshold)
    - Significant operational changes
    - Time threshold exceeded (configurable, default 30 days)
  Output: RetrainDecision { ShouldRetrain, Reason, Severity }
          Domain Service — stateless, no persistence

CELL-5: AHS.Cell.Xinfer.ModelTrainer
  Purpose: Retrain when RetrainDecider approves
  Input: HistoricalDataset + ShipmentFeaturesSchema
  Technology: ML.NET (AOT-compatible FastTree MART)
  Output: ModelVersion(n+1) sealed in GxP Ledger
  GxP: Every retrain is a SignedCommand + Ledger entry

CELL-6: AHS.Cell.Xinfer.ModelPredictor
  Purpose: Predict risk using active model
  Input: ShipmentDTO + ModelVersion
  Output: PredictionResult {
    Probability,
    RiskLevel,        // Low | Medium | High | Critical
    FeatureContributions[]  // XAI DNA — 14 factors
  }
  Performance: P99 < 10ms (existing SIMD engine preserved)

CELL-7: AHS.Cell.Xinfer.Recommender
  Purpose: Generate actionable recommendations
  Input: PredictionResult + HistoricalData + OperationalFactors
  Output: Recommendations[] {
    Action,           // "Change carrier to ACTIVE_REEFER"
    Priority,         // Critical | High | Medium | Low
    ExpectedRiskDelta // -15% if action taken
  }
  Implementation: Rule-based for MVP (GxP auditable)
                  Rules documented as domain constants
```

## SECTION 3 — INTER-CELL FLOW

```csharp
// Orchestration flow (in AHS.Web.Hive BFF or API Gateway):

// Step 1 — Capture
var shipment = await ShipmentInputAdapter.CaptureAsync(request);

// Step 2 — Validate
var readiness = await DataReadinessService.EvaluateAsync(shipment);
if (readiness.Status == ReadinessStatus.NotAcceptable)
    return Results.BadRequest(readiness.Errors);

// Step 3 — Select historical data
var historical = await HistoricalSelector.SelectAsync(shipment);

// Step 4 — Decide retrain
var retrainDecision = RetrainDecider.Evaluate(historical, currentModel);
if (retrainDecision.ShouldRetrain)
    await ModelTrainer.TrainAsync(historical, signedBy, reason); // GxP

// Step 5 — Predict
var prediction = await ModelPredictor.PredictAsync(shipment, activeModel);

// Step 6 — Recommend
var recommendations = await Recommender.GenerateAsync(prediction, historical);

// Step 7 — Seal in GxP Ledger
await Ledger.AppendAsync(new XinferRiskAssessed(
    shipment, prediction, recommendations, signedBy, reason));

// Step 8 — Publish to Hive
await Publisher.PublishAsync(new XinferRiskAssessed(...));
```

## SECTION 5 — GxP REQUIREMENTS

```
Every state-changing operation across all 7 Cells requires SignedCommand:

CELL-5 ModelTrainer:
  TrainModelCommand : SignedCommand
  → Reason: "Retraining triggered by [RetrainDecision.Reason]"
  → Ledger: ModelRetrained event with ModelVersion sealed

CELL-6 ModelPredictor:
  PredictRiskCommand : SignedCommand
  → Reason: required for every prediction used in a shipping decision
  → Ledger: XinferRiskAssessed event

CELL-7 Recommender:
  ApplyRecommendationCommand : SignedCommand
  → Reason: required when operator acts on a recommendation
  → Ledger: RecommendationApplied event

READ operations (DataReadiness, HistoricalSelector, RetrainDecider):
  → No SignedCommand required (read-only, no state change)
  → But results are included in the sealed Ledger entry of Step 7
```
